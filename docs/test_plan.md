# Тест‑план (упрощённый)

## 1. Область тестирования
- Веб‑приложение Flask: аутентификация, объявления, карточка, сделки (обмен/пожертвование/переработка), справка, REST API (категории, объявления).
- База данных MySQL: целостность данных, миграции/инициализация.

## 2. Виды тестирования
- Функциональное тестирование UI (позитивные/негативные сценарии).
- Тестирование API (статусы, схема данных, права доступа).
- Тестирование прав и ролей (admin/manager/client).
- Установочное тестирование (пошаговый запуск по user_guide).

## 3. Критерии начала/окончания
- Начало: установлен Python и MySQL, настроен .env, выполнена инициализация БД.
- Окончание: пройдены ключевые сценарии без критических дефектов; API возвращает корректные ответы; данные сохраняются.

## 4. Окружение
- ОС: Windows 10/11.
- Python 3.8+.
- MySQL 8.x.
- Браузер: Edge/Chrome.

## 5. Риски
- Отличия версий библиотек (совместимость).
- Ошибки прав доступа в MySQL.
- Нарушение схемы при ручной модификации БД.

## 6. Что не тестируем
- Мобильная адаптация (базовый Bootstrap достаточен).
