# Тест‑кейсы (примерные)

## TC-001 Регистрация пользователя (позитив)
- Предусловия: сервер запущен, БД инициализирована.
- Шаги:
  1) Открыть /auth/register
  2) Ввести username, email, пароль и подтверждение
  3) Нажать «Зарегистрироваться»
- Ожидаемый результат: редирект на /auth/login, флеш «Регистрация успешна».

## TC-002 Вход в систему (позитив)
- Шаги: /auth/login → ввести валидные логин/пароль → «Войти»
- ОР: редирект в «Личный кабинет».

## TC-003 Создание объявления (позитив)
- Шаги: Личный кабинет → «Добавить объявление» → заполнить форму → Сохранить
- ОР: флеш «Объявление опубликовано», карточка объявления открыта.

## TC-004 Фильтрация объявлений (позитив)
- Шаги: /items → выбрать категорию и статус → Показать
- ОР: список сокращается в соответствии с фильтрами.

## TC-005 Модалка «Пожертвовать» (негатив без прав)
- Предусловия: пользователь A, вещь пользователя B
- Шаги: открыть карточку вещи B → попытаться пожертвовать
- ОР: кнопка недоступна; POST вернёт 403 при обходе.

## TC-006 Редактирование справки (негатив без роли admin)
- Шаги: /admin/help без роли admin
- ОР: 403 Forbidden.

## TC-007 REST API: GET /api/items (позитив)
- Шаги: авторизоваться → GET /api/items
- ОР: 200 OK, JSON со списком вещей (до 100 позиций), корректные поля.

## TC-008 REST API: POST /api/items (негатив)
- Шаги: авторизоваться → POST /api/items без обязательных полей
- ОР: 400 Bad Request c сообщением об отсутствующих полях.
